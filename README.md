## 🌟 نبذة عن المشروع
**تدبر** هي منصة متكاملة صُممت لتكون عملاً تقنياً احترافياً وصدقة جارية تهدف لخدمة كتاب الله. تم بناء المنصة لتوفر تجربة بحث وتدبر معاصرة، من خلال تحليل بنية القرآن واستخراج العلاقات بين السور والمواضيع باستخدام **قواعد البيانات الرسومية (Neo4j)**، إلى جانب عرض القراءات، والتفاسير، وأحكام التجويد في واجهة مستخدم (UI) فخمة ومريحة للعين تعتمد على الألوان الهادئة المستوحاة من المخطوطات القرآنية.

---

## 🚀 المميزات الرئيسية

- **الرسم البياني للمفاهيم (Knowledge Graph):** تصور تفاعلي مبتكر للروابط الموضوعية بين السور والمواضيع داخل القرآن.
- **تصفح القرآن الذكي:** عرض احترافي للسور مع إمكانية الوصول السريع، والبحث اللحظي المتقدم، وعرض العناوين والمواضيع الفرعية لكل سورة.
- **قاعدة معرفية واسعة:** أقسام مخصصة لعلوم القرآن (آيات الأحكام، الناسخ والمنسوخ، الأمثال القرآنية، التشريعات، المتشابهات).
- **تصميم متجاوب وحديث:** واجهة مستخدم مبنية بـ React توفر الوضع الليلي/النهاري (Dark/Light mode) مع التوافق التام مع الشاشات المختلفة.
- **استخراج البيانات آلياً (Scraping Pipelines):** دعم شامل لسحب وتحليل وتجهيز البيانات القرآنية من مصادر موثوقة باستخدام Playwright و Python.

---

## 🛠️ التقنيات المستخدمة (Tech Stack)

### 🎨 الواجهة الأمامية (Frontend)
- **React.js**: مكتبة بناء واجهات المستخدم.
- **Tailwind CSS**: إطار عمل التصميم (التنسيقات والألوان).
- **Framer Motion**: الحركات والانتقالات (Animations).
- **Lucide React**: مكتبة الأيقونات.

### ⚙️ الواجهة الخلفية وقواعد البيانات (Backend & Databases)
- **FastAPI**: إطار عمل خفيف وسريع لبناء خدمات الـ APIs.
- **Neo4j**: قاعدة بيانات رسومية (Graph DB) لتخزين العلاقات والمواضيع المعقدة.
- **PostgreSQL**: قاعدة بيانات علائقية لتخزين نصوص الآيات والمواضيع الفرعية.
- **Playwright & BeautifulSoup**: لاستخراج وتحليل البيانات القرآنية آلياً (Scraping).

## 💻 كيفية التشغيل محلياً (Local Setup)

### 1. المتطلبات الأساسية
تأكد من تثبيت البرامج التالية على جهازك:
- [Node.js](https://nodejs.org/) (لـ Frontend)
- [Python 3.10+](https://www.python.org/) (لـ Backend)
- قاعدة بيانات [PostgreSQL](https://www.postgresql.org/)
- قاعدة بيانات [Neo4j Desktop / Aura](https://neo4j.com/)

### 2. تشغيل الواجهة الخلفية (Backend)
```bash
# الانتقال إلى مجلد مسار البيانات والخادم
cd tadabbur-data

# تثبيت الحزم المطلوبة
pip install -r requirements.txt

# الذهاب لمجلد السكربتات
cd Scripts

# تشغيل الخادم
uvicorn main:app --reload
```
سيعمل الخادم على الرابط: `http://localhost:8000`

### 3. تشغيل الواجهة الأمامية (Frontend)
```bash
# الانتقال إلى مجلد الواجهة الأمامية في نافذة أوامر جديدة (Terminal)
cd tadabbur-ui

# تثبيت الحزم
npm install

# تشغيل التطبيق
npm start
```
ستعمل الواجهة على الرابط: `http://localhost:3000`
